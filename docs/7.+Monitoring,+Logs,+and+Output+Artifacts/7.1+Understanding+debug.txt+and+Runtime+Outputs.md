# 7 Monitoring, Logs, and Output Artifacts

## 7.1 Understanding debug.txt and Runtime Outputs

This application generates several runtime artifacts and log files that provide insight into its execution, track intermediate data (image lists, audio feature timings), and capture errors or configuration details. Understanding these files is essential for troubleshooting, customizing behavior, or integrating output downstream.

### 7.1.1 debug.txt (Run Log)

Location

• Created in the current working directory at the very start of execution by opening a `std::ofstream` on `"debug.txt"`.

Generation and Content

• The `_tmain` function calls

```cpp
myofstream.open("debug.txt", std::ios::out);
if (myofstream.is_open())
{
    myofstream << "spifractaltraceanimaudiobnopcrossfadegeneralized_vs2017.exe" << endl << endl;
    myofstream << "nArgs is :" << nArgs << endl << endl;
    myofstream << "modestring is: " << global_modestring << endl;
    myofstream << "global_debug is: " << global_debug << endl;
    myofstream << "global_INPUTIMAGES_ARE_VIDEOSEQUENCEFRAMES is: " << global_INPUTIMAGES_ARE_VIDEOSEQUENCEFRAMES << endl;
    myofstream << "global_INPUTIMAGES_ARE_TOBEUSEDASSEQUENCEOFFRAMES is: " << global_INPUTIMAGES_ARE_TOBEUSEDASSEQUENCEOFFRAMES << endl;
    // … additional global flags and parameters …
    myofstream.flush();
}
```

as the very first actions in `_tmain` to record command-line parameters and mode flags .

• Subsequent calls to `myofstream << ...` log status messages (e.g., directory creation, audio file errors), error conditions, and segmentations, providing a chronological trace of the application’s internal decisions and any failures.

• Sample contents:

```plaintext
spifractaltraceanimaudiobnopcrossfadegeneralized_vs2017.exe

nArgs is : 24

modestring is: 
global_debug is: 0
global_INPUTIMAGES_ARE_VIDEOSEQUENCEFRAMES is: 0
global_INPUTIMAGES_ARE_WIDESCREENFRAMES is: 0
global_INPUTIMAGES_ARE_TOBESCALEDDOWNTOIDENTICALSIZE is: 0

error, can't open audio file
```

### 7.1.2 spifractaltrace_filenames.txt (Input Image Enumeration)

Location

• Written to the working directory immediately after parsing command-line arguments.

Generation

```cpp
string systemcommand = "DIR \"" + global_imagefolder + "\\*" + global_imageextension
    + "\" /B /S /O:N > spifractaltrace_filenames.txt";
system(systemcommand.c_str());
```

• This issues a Windows `DIR` command to list all files matching the configured extension under the input folder, sorting by name, and redirects output into `spifractaltrace_filenames.txt` .

Content and Interpretation

• Each line is the full path to one input image (e.g., `D:\images\frame_000001.jpg`).

• Read back into the application to populate the `global_imagefilenames` vector, driving which images are used for fractal-trace transformations.

### 7.1.3 aubiotrack_beattimes.txt (Beat Times)

Location

• Written to the working directory after image enumeration.

Generation

```cpp
string systemcommand = global_aubiotrackpath + " \"" + global_audiofilename
    + "\" > aubiotrack_beattimes.txt";
system(systemcommand.c_str());
```

• Invokes the Aubio CLI to detect beat onsets and writes one timestamp per line in seconds .

Content and Interpretation

• Lines contain floating-point values, e.g.:

```plaintext
  0.523487
  1.047365
  1.572246
  …  
```

• Loaded into `global_audiobeattimes_sec`; used to determine at which audio-beat frames should change.

### 7.1.4 aubiotrack_notetimes.txt (Note Times)

Location

• Generated when note-based timing (`bnop="note"`) is enabled.

Generation

```cpp
string systemcommand = global_aubionotespath + " \"" + global_audiofilename
    + "\" > aubiotrack_notetimes.txt";
system(systemcommand.c_str());
```

• Invokes Aubio’s note-detection CLI to produce tab-delimited lines with three fields .

Content and Interpretation

• Each line format:

```plaintext
  <MIDI_note>    <time_seconds>    <strength_or_other>
```

• The code parses the first two fields into `global_audionotetimes_midinote` and `global_audionotetimes_sec`.

• Subsequent segmentation (quantization) of note times drives when frames change on significant pitch shifts.

### 7.1.5 Other Aubio Outputs

Depending on the `bnop` mode, the application may also generate:

• **aubiotrack_pitchtimes.txt** via `global_aubiopitchpath` (pitch detection)

• **aubiotrack_onsettimes.txt** via `global_aubioonsetpath` (onset detection)

• **aubiotrack_pitchtimes.txt** and **aubiotrack_onsettimes.txt** follow the same pattern: one timestamp (or value and timestamp) per line, parsed into corresponding global vectors for frame-timing decisions.

### 7.1.6 Generated Frame Files

Location

• Under the output folder (`global_outputimagefolder`), typically within a session-specific subfolder.

Naming Convention

• Files saved by:

```cpp
FreeImage_Save(FIF_JPEG, global_pNew24bitDIB, finalnewfilename.c_str());
```

• Where `finalnewfilename` is constructed as:

```plaintext
<framefolder>\<framefilenameprefix><zero-padded-frame-index><videofilenameext>
```

(e.g., `frames\frame_000123.jpg`) .

Interpretation

• Each JPEG represents one fractal-traced frame corresponding to a segment of the audio track.

• Review these to verify correct timing alignment and visual output before video encoding or crossfading.

### 7.1.7 Final MP4 Video

Location

• Written to the output folder (often alongside the frame subfolder).

Generation

```cpp
string systemcommand;
if (global_mergeaudiowithfinalvideo)
{
  systemcommand = global_ffmpegpath + " -i \"" + global_audiofilename + "\" -r " + bufferfps
      + " -s " + bufferscale + " -start_number 1 -i \"" + finalnewframefolder + "\\" + global_framefilenameprefix
      + "%6d" + global_framefilenameext + "\" -vcodec libx264 -crf 25 -pix_fmt yuv420p \""
      + finalvideofilename + "\"";
}
else
{
  systemcommand = global_ffmpegpath + " -r " + bufferfps
      + " -s " + bufferscale + " -start_number 1 -i \"" + finalnewframefolder + "\\" + global_framefilenameprefix
      + "%6d" + global_framefilenameext + "\" -vcodec libx264 -crf 25 -pix_fmt yuv420p \""
      + finalvideofilename + "\"";
}
system(systemcommand.c_str());
```

• Uses FFmpeg to wrap the sequence of JPEG frames into an H.264-encoded MP4 at the configured frame rate and resolution. If `global_mergeaudiowithfinalvideo` is true, the original audio track is muxed into the output video .

Artifacts

• `<session>(with-audio).mp4` or `<session>.mp4` containing the animated fractal trace synchronized to audio.

• Suitable for playback or further editing.
