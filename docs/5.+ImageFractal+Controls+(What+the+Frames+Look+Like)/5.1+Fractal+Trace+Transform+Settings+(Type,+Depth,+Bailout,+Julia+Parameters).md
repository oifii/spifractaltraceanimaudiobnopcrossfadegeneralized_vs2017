# 5. Image/Fractal Controls (What the Frames Look Like)

## 5.1 Fractal Trace Transform Settings (Type, Depth, Bailout, Julia Parameters)

The Fractal Trace Transform remaps each pixel of the input image through an iterative fractal “escape-time” process, producing swirling, self-similar patterns that evolve over successive frames. Users tune four core parameters—type, depth, bailout threshold, and Julia seeds—to control the mathematical mapping and thus the visual complexity, color flow, and degree of distortion in each generated frame.

### Global Parameter Definitions

All settings are exposed as global variables in `spifractaltrace*.cpp` and configured via command-line arguments in `_tmain`. Their defaults and comments appear here:

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| global_fractaltrace_type | int | 1 | Which equation to apply:<br/>0 = Mandelbrot, 1 = Julia, 2 = Julia-Mandelbrot |
| global_fractaltrace_depth | int | 3 | Number of iterations per pixel (trace depth). Higher values add detail at the cost of speed |
| global_fractaltrace_bailout | double | 10000.0 | Bail-out radius (squared) for Julia equations. Smaller values force earlier escape and coarser patterns |
| global_fractaltrace_jx | double | –0.5 | Julia seed constant X component (only used for Julia and Julia-Mandelbrot) |
| global_fractaltrace_jy | double | 0.75 | Julia seed constant Y component |


#### Fractal Type Constants

```cpp
#define FRACTALTRACE_TYPE_MANDELBROT        0
#define FRACTALTRACE_TYPE_JULIA             1
#define FRACTALTRACE_TYPE_JULIA_MANDELBROT  2
```

(default `global_fractaltrace_type = FRACTALTRACE_TYPE_JULIA`)

### How the Switch in DoFractalTrace Applies These Settings

Inside `DoFractalTrace`, the code maps each pixel coordinate into fractal space, then invokes one of three routines based on `global_fractaltrace_type`:

```cpp
switch (global_fractaltrace_type)
{
    case FRACTALTRACE_TYPE_MANDELBROT:
        mandelbrot(cx, cy, &px, &py);
        break;
    case FRACTALTRACE_TYPE_JULIA:
        julia(cx, cy, global_fractaltrace_jx, global_fractaltrace_jy, &px, &py, bailout2);
        break;
    case FRACTALTRACE_TYPE_JULIA_MANDELBROT:
        julia(cx, cy, cx, cy, &px, &py, bailout2);
        break;
    default:
        mandelbrot(cx, cy, &px, &py);
}
```

This snippet appears in several variants of `spifractaltraceanimaudiobnopcrossfade…` and in the core console entry point .

- **Mandelbrot**

Calls `mandelbrot(x, y, &u, &v)`, which iterates a standard Mandelbrot loop exactly `global_fractaltrace_depth` times, ignoring bailout checks.

- **Julia**

Calls `julia(x, y, jx, jy, &u, &v, bailout2)`, iterating until either `depth` is reached or the point “escapes” beyond the squared radius `bailout2`.

- **Julia-Mandelbrot**

Seeds each iteration with the original coordinate `(cx, cy)` (rather than the fixed constants), blending Julia and Mandelbrot behaviors.

### Julia Function and Bail-Out Logic

The Julia routine used above implements the escape-time algorithm:

```cpp
void julia(double x, double y, double jx, double jy, double *u, double *v, double bailout2)
{
    int i;
    double xx = x;
    double yy = y;
    for (i = 0; i < global_fractaltrace_depth; i++)
    {
        double x2 = xx * xx;
        double y2 = yy * yy;
        double tmp = x2 - y2 + jx;
        yy = 2 * xx * yy + jy;
        xx = tmp;
        if ((x2 + y2) > bailout2)
            break;
    }
    *u = xx;
    *v = yy;
}
```

- `global_fractaltrace_depth` controls the maximum loop count.
- `bailout2` is computed once per row as `global_fractaltrace_bailout * global_fractaltrace_bailout`.
- The loop exits early when the point’s squared magnitude exceeds `bailout2`, producing sharp color transitions near the escape boundary .

---

By fine-tuning these four parameters—**type**, **depth**, **bailout**, and the Julia seeds (**jx**, **jy**)—artists can explore a vast palette of fractal distortions, from smooth swirls to dense filaments and glitch-like jumps, driving the aesthetic of each audio-synchronized frame.
