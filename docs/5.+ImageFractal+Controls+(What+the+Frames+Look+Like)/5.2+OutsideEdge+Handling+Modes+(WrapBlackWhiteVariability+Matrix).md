## 5.2 Outside/Edge Handling Modes (Wrap / Black / White / Variability Matrix)

When the fractal‐trace mapping (`DoFractalTrace`) computes a source coordinate (px, py) that falls outside the bounds of the input image, the algorithm dispatches to one of four “outside‐type” modes controlled by the global variable `global_fractaltrace_outsidetype`. The modes and their numeric values are defined as follows:

```cpp
#define OUTSIDE_TYPE_WRAP                        0
#define OUTSIDE_TYPE_BLACK                       1
#define OUTSIDE_TYPE_WHITE                       2
#define OUTSIDE_TYPE_WRAP_WITHVARIABILITYMATRIX  3  // uses a VARIABILITYMATRIXSIZE×VARIABILITYMATRIXSIZE context
#define VARIABILITYMATRIXSIZE                    9  // must be odd, default 9×9 matrix
int global_fractaltrace_outsidetype = 0;  // default: WRAP 
```

Inside `DoFractalTrace`, after computing and normalizing `(px, py)`, this switch handles out‐of‐bounds coordinates:

```cpp
if (px < 0 || px >= imagewidth || py < 0 || py >= imageheight)
{
    switch (global_fractaltrace_outsidetype)
    {
        case OUTSIDE_TYPE_WRAP:
            // ...
            break;
        case OUTSIDE_TYPE_WRAP_WITHVARIABILITYMATRIX:
            // ...
            break;
        case OUTSIDE_TYPE_BLACK:
            // ...
            break;
        case OUTSIDE_TYPE_WHITE:
            // ...
            break;
    }
}
```

### 5.2.1 OUTSIDE_TYPE_WRAP

Wraps coordinates back into the input image via modulo arithmetic, then samples bilinearly from the original image.

Behavior:

- `px = fmod(px, imagewidth); py = fmod(py, imageheight);`
- If negative, add the image dimension to bring into [0, size).
- Call `pixels_get_biliner(input_dib, px, py, &pixel)`

```cpp
case OUTSIDE_TYPE_WRAP:
    px = fmod(px, imagewidth);
    py = fmod(py, imageheight);
    if (px < 0.0) px += imagewidth;
    if (py < 0.0) py += imageheight;
    pixels_get_biliner(input_dib, px, py, &pixel);
    break;
```

### 5.2.2 OUTSIDE_TYPE_BLACK

Fills out‐of‐bounds pixels with opaque black.

Behavior:

- `pixel.rgbRed = pixel.rgbGreen = pixel.rgbBlue = 0;`
- Alpha remains at 255 (fully opaque) or as originally set.

```cpp
case OUTSIDE_TYPE_BLACK:
    pixel.rgbRed   = pixel.rgbGreen = pixel.rgbBlue = 0;
    break;
```

### 5.2.3 OUTSIDE_TYPE_WHITE

Fills out‐of‐bounds pixels with opaque white.

Behavior:

- `pixel.rgbRed = pixel.rgbGreen = pixel.rgbBlue = 255;`
- Alpha remains at 255.

```cpp
case OUTSIDE_TYPE_WHITE:
    pixel.rgbRed   = pixel.rgbGreen = pixel.rgbBlue = 255;
    break;
```

### 5.2.4 OUTSIDE_TYPE_WRAP_WITHVARIABILITYMATRIX

In this mode, instead of wrapping around a single input image, the algorithm uses a *variability matrix* of neighboring frames to introduce temporal or contextual variation. A global 2D array

```cpp
int global_vm[VARIABILITYMATRIXSIZE][VARIABILITYMATRIXSIZE];
map<FIBITMAP*, int> global_imagehandlesmap;
vector<FIBITMAP*>   global_imagehandles;
```

defines, for each input frame handle, offsets into the image‐handles vector .

Behavior when `(px, py)` is out of bounds:

1. Normalize parameters:

```cpp
   double px_param = px / imagewidth, py_param = py / imageheight;
   double px_fractpart = modf(px_param, &px_intpart);
   double py_fractpart = modf(py_param, &py_intpart);
```

1. Compute a variability‐matrix index around center (`vm_edgesize = (9−1)/2 = 4`):

```cpp
   int indexpx    = (int)px_intpart % (vm_edgesize + 1);
   int indexpy    = (int)py_intpart % (vm_edgesize + 1);
   int vm_indexpx = indexpx + vm_edgesize;
   int vm_indexpy = indexpy + vm_edgesize;
```

1. Lookup the base input‐frame index, add the matrix offset, and wrap within available frames:

```cpp
   auto it = global_imagehandlesmap.find(input_dib);
   int inputframe_index     = it->second;
   int alternateframe_index = (inputframe_index
                              + global_vm[vm_indexpx][vm_indexpy])
                              % (int)global_imagehandles.size();
   FIBITMAP* alternate_dib  = global_imagehandles[alternateframe_index];
```

1. Wrap pixel coordinates within the chosen frame and sample:

```cpp
   px = fmod(px, imagewidth);
   py = fmod(py, imageheight);
   if (px < 0.0) px += imagewidth;
   if (py < 0.0) py += imageheight;
   if (alternate_dib)
       pixels_get_biliner(alternate_dib, px, py, &pixel);
   else
       pixels_get_biliner(input_dib,     px, py, &pixel);
```

1. Fall back to the original image if `alternate_dib` is null.

```cpp
case OUTSIDE_TYPE_WRAP_WITHVARIABILITYMATRIX:
{
    // compute vm_indexpx, vm_indexpy as above...
    map<FIBITMAP*, int>::iterator it = global_imagehandlesmap.find(input_dib);
    int inputframe_index = it->second;
    int af_index = (inputframe_index
                   + global_vm[vm_indexpx][vm_indexpy])
                   % (int)global_imagehandles.size();
    FIBITMAP* af_dib = global_imagehandles.at(af_index);

    px = fmod(px, imagewidth);
    py = fmod(py, imageheight);
    if (px < 0.0) px += imagewidth;
    if (py < 0.0) py += imageheight;

    if (af_dib)
        pixels_get_biliner(af_dib, px, py, &pixel);
    else
        pixels_get_biliner(input_dib, px, py, &pixel);
}
break;
```

---

With these modes, the transform can either seamlessly tile the input (`WRAP`), pad with solid colors (`BLACK`/`WHITE`), or introduce cross‐image variability for rich, context‐aware fractal tracing (`WRAP_WITHVARIABILITYMATRIX`).
