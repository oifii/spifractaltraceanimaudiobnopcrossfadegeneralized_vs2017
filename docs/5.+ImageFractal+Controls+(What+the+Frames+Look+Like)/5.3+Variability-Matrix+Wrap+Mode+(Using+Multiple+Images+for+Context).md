# 5.3 Variability-Matrix Wrap Mode (Using Multiple Images for Context)

## Overview

The Variability-Matrix Wrap Mode extends the standard “wrap” outside-handling by sampling from multiple input frames arranged in a logical 2D context matrix. Instead of always wrapping back onto the same source image when a mapped coordinate (`px`,`py`) falls outside its bounds, the algorithm:

1. Divides the extrapolated fractal-trace coordinates into discrete “tiles” around a central cell.
2. Uses a pre-populated matrix of random frame-index offsets (`global_vm`) to choose an alternate source image for each tile.
3. Wraps within that alternate image’s dimensions and performs bilinear sampling.

This produces richer, multi-image collage effects in the per-pixel trace, giving each area of the fractal-mapped output the “context” of neighboring frames.

## Key Data Structures

| Identifier | Type | Description |
| --- | --- | --- |
| VARIABILITYMATRIXSIZE | `#define` (int) | Must be odd (≥ 3). Defines the matrix dimensions. Default is 9 → 9×9 tiles around center . |
| global_imagehandlesmap | `map<FIBITMAP*,int>` | Maps each loaded `FIBITMAP*` pointer to its index in `global_imagehandles`. Used to find the “base” frame index for the current input image . |
| global_vm | `int[VARIABILITYMATRIXSIZE][VARIABILITYMATRIXSIZE]` | A matrix of random index-offsets into `global_imagehandles`. Each cell holds a value in [0…N–1], where N is the number of loaded frames . |


## Initialization of Image Handles and Variability Matrix

Before any fractal tracing begins, the helper function `reloadallimages_and_populateimagehandles()` loads all selected input images into memory and builds both `global_imagehandlesmap` and `global_vm`:

```cpp
// After loading all images into global_imagehandles:
global_imagehandlesmap.clear();
// Populate map: each FIBITMAP* → its index
for (int i = 0; i < global_imagehandles.size(); ++i) {
    global_imagehandlesmap.insert({ global_imagehandles[i], i });
}
// Populate variability matrix with random offsets [0 .. N-1]
int min_offset = 0;
int max_offset = global_imagehandles.size() - 1;
for (int ix = 0; ix < VARIABILITYMATRIXSIZE; ++ix) {
  for (int iy = 0; iy < VARIABILITYMATRIXSIZE; ++iy) {
    global_vm[ix][iy] = RandomInt(min_offset, max_offset);
  }
}
```

This code appears in the implementation of `reloadallimages_and_populateimagehandles()` .

## Fractal Trace: Selecting an Alternate Frame

Within `DoFractalTrace(...)`, when `global_fractaltrace_outsidetype == OUTSIDE_TYPE_WRAP_WITHVARIABILITYMATRIX`, the pixel-sampling logic performs the following steps for out-of-bounds coordinates:

```cpp
case OUTSIDE_TYPE_WRAP_WITHVARIABILITYMATRIX: {
    // 1) Normalize to “tile” coordinates
    double px_param = px / imagewidth;
    double py_param = py / imageheight;
    double px_fract, py_fract, px_int, py_int;
    px_fract = modf(px_param, &px_int);
    py_fract = modf(py_param, &py_int);

    // 2) Compute matrix indices around center
    int vm_edgesize = (VARIABILITYMATRIXSIZE - 1) / 2;
    int indexpx = ((int)px_int) % (vm_edgesize + 1);
    int indexpy = ((int)py_int) % (vm_edgesize + 1);
    int vm_indexpx = indexpx + vm_edgesize;
    int vm_indexpy = indexpy + vm_edgesize;

    // 3) Get base frame index for this input_dib
    auto it = global_imagehandlesmap.find(input_dib);
    int base_index = it->second;

    // 4) Compute alternate frame index via variability matrix
    int offset = global_vm[vm_indexpx][vm_indexpy];
    int alt_index = (base_index + offset) % global_imagehandlesmap.size();
    FIBITMAP* alt_dib = global_imagehandles.at(alt_index);

    // 5) Wrap px,py into [0..width)×[0..height) and sample
    px = fmod(px, imagewidth); if (px < 0) px += imagewidth;
    py = fmod(py, imageheight); if (py < 0) py += imageheight;
    if (alt_dib)
        pixels_get_biliner(alt_dib, px, py, &pixel);
    else
        pixels_get_biliner(input_dib, px, py, &pixel);
} break;
```

This logic is implemented in the `DoFractalTrace` function’s variability-matrix branch .

### Flowchart: Variability-Matrix Frame Lookup

```mermaid
flowchart LR
    A[Compute px, py - fract traced] --> B[Normalize: px_param = px/width, py_param = py/height]
    B --> C[Split into integer & fractional parts via modf]
    C --> D[Compute vm_indexpx, vm_indexpy around center: edgesize = SIZE-1 / 2]
    D --> E[Lookup cell offset = global_vm at vm_indexpx vm_indexpy]
    E --> F[Find base_index = global_imagehandlesmap at input_dib]
    F --> G[alt_index = base_index + offset mod total_frames]
    G --> H[alt_dib = global_imagehandles at alt_index]
    H --> I[Wrap px,py into 0,width × 0,height]
    I --> J[Sample pixel from alt_dib at wrapped px,py]
    J --> K[Write pixel to output_dib]
```

---

By leveraging a pre-randomized matrix of frame-index offsets, the Variability-Matrix Wrap Mode blends neighboring input images in a spatially coherent fashion, enriching the visual texture of the fractal trace animation without complex per-frame logic.
