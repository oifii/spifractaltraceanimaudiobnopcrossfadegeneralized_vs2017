## 8. Troubleshooting & FAQs

### 8.1 Common Failures and Fixes (No Images, No Aubio Events, ffmpeg Errors)

When running the `spifractaltrace…` tools you may encounter three frequent failure modes. The checks and remedies below refer directly to the `DIR`-based image discovery, Aubio CLI invocations, and FFmpeg calls in the code.

---

#### No Images Found

Symptoms:

- `spifractaltrace_filenames.txt` is created but empty.
- No frames are generated (no image handles loaded).

Checks & Fixes:

- **Verify folder path and extension filter**

The code builds the filter as:

```cpp
  pathfilter = global_imagefolder + "\\*" + global_imageextension;
  systemcommand = "DIR " + quote + pathfilter + quote + "/B /S /O:N > spifractaltrace_filenames.txt";
  system(systemcommand.c_str());
```

• Ensure `global_imagefolder` is the correct absolute or relative path.

• Ensure `global_imageextension` includes the leading dot (e.g. `.png`, `.bmp`). A missing or extra dot produces `*png` or `*..png`, which matches nothing.

• Run the printed `DIR` command manually in a Command Prompt to confirm it lists files.

- **Check working directory and permissions**

• The tool writes `spifractaltrace_filenames.txt` in the current working directory. Make sure you have write permission and you’re running from the expected folder.

• If you see “File Not Found” in the console, adjust the working directory or use an absolute `global_imagefolder`.

---

#### No Aubio Events

Symptoms:

- `aubiotrack_beattimes.txt` (and similarly for notes/pitch/onsets) is empty or missing.
- No beat/note/onset/pitch vectors are populated, so no image changes occur.

Checks & Fixes:

- **Validate Aubio executable path**

Invoked as:

```cpp
  systemcommand = global_aubiotrackpath + " " + quote + global_audiofilename + quote + " > aubiotrack_beattimes.txt";
  system(systemcommand.c_str());
```

• Confirm `global_aubiotrackpath` (and similarly `global_aubionotespath`, `global_aubiopitchpath`, `global_aubioonsetpath`) points to the correct `.exe`.

• If the path contains spaces, wrap only the executable in quotes:

```cpp
    systemcommand = quote + global_aubiotrackpath + quote
                  + " " + quote + global_audiofilename + quote
                  + " > aubiotrack_beattimes.txt";
```

- **Ensure input audio is a valid WAV**

• After FFmpeg conversion to `mono_audiofilename.wav`, make sure this file exists and is a mono WAV. If you modified the code to feed Aubio the mono file, update the invocation accordingly.

- **Inspect the output file**

• Open `aubiotrack_beattimes.txt` in a text editor. If it is empty or contains an error message, run the same command manually in CMD to see the error – often missing DLLs or unsupported sample rate.

---

#### FFmpeg Errors

Symptoms:

- `mono_audiofilename.wav` is not created.
- Console prints “’ffmpeg’ is not recognized” or reports codec/permission errors.
- Final MP4 wrapping fails or produces no video.

Checks & Fixes:

- **Validate FFmpeg path and quoting**

The conversion step is coded as:

```cpp
  systemcommand = global_ffmpegpath + " -y -i "
                + quote + global_audiofilename + quote
                + " -ac 1 mono_audiofilename.wav";
  systemcommand = quote + systemcommand + quote;
  system(systemcommand.c_str());
```

• If `global_ffmpegpath` contains spaces, wrap only the executable, not the entire command:

```cpp
    systemcommand = quote + global_ffmpegpath + quote
                  + " -y -i " + quote + global_audiofilename + quote
                  + " -ac 1 mono_audiofilename.wav";
```

• Avoid double-wrapping the whole command in quotes, which Windows CMD may treat as a single literal.

- **Check printed command and run manually**

• The code prints `systemcommand` before execution. Copy that line into a shell to observe FFmpeg’s diagnostics.

- **Verify return code and output**

• If FFmpeg runs but the WAV is invalid, inspect console for “Invalid data found” or similar.

• Ensure you have write permission in the working directory and sufficient disk space.

- **Wrapping frames into MP4**

When merging frames and audio:

```cpp
  systemcommand = global_ffmpegpath
                + " -r " + bufferfps
                + " -s " + bufferscale
                + " -start_number 1 -i "
                + quote + framePattern + quote
                + " -vcodec libx264 -crf 25 -pix_fmt yuv420p "
                + finalVideofilename;
  system(systemcommand.c_str());
```

• Confirm your `%6d` pattern matches the actual frame filenames.

• If audio merge fails, check that `-i` for audio is correctly ordered before `-i` for frames.

---

By systematically verifying paths, quoting only as needed, and running the exact printed commands in a shell, you can isolate and resolve most “no output” failures in image discovery, Aubio analysis, and FFmpeg processing.
