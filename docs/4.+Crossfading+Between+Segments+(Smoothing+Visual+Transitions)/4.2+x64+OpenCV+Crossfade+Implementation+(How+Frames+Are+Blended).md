## 4.2 x64 OpenCV Crossfade Implementation

In the 64-bit build, frame crossfades are performed in‐process using OpenCV’s image I/O and blending routines rather than invoking ImageMagick. In each crossfade segment, pairs of frames straddling a beat/note/onset boundary are loaded, blended with linearly varying alpha weights, and the result is written back to disk, overwriting the “first” frame in the pair. This produces a smooth dissolve between the end of one segment and the start of the next.

### Frame Pair Identification

Within the loop over `numberofframeforthiscrossfade`, each iteration computes:

- `framenumber_first` – the ascending frame index from the end of the preceding segment.
- `framenumber_second` – the descending frame index from the start of the following segment.
- `fpercent_first` – the percentage weight for the first frame:

```cpp
  float fpercent = 100.0f/(numberofframeforthiscrossfade+1.0f);
  float fpercent_first  = fpercent*(index+1);
  float fpercent_second = 100.0f - fpercent_first;
```

These percentages are then converted to fractions (`0.0f`–`1.0f`) for blending .

The two filenames are constructed as:

```cpp
char buf[256];
// First segment frame
sprintf(buf, "%06d", framenumber_first);
string finalnewfilename_first  = finalnewframefolder + "\\" + newframefilenameprefix + buf + global_imageextension;
// Second segment frame
sprintf(buf, "%06d", framenumber_second);
string finalnewfilename_second = finalnewframefolder + "\\" + newframefilenameprefix + buf + global_imageextension;
```

### Image Loading and Validation

The two images are loaded into OpenCV mats:

```cpp
cv::Mat src1 = cv::imread(finalnewfilename_first.c_str());
cv::Mat src2 = cv::imread(finalnewfilename_second.c_str());
```

Each `cv::imread` may return an empty `cv::Mat` if the file is missing or unreadable. The implementation only proceeds when **both** `src1` and `src2` are non‐empty:

```cpp
if (!src1.empty() && !src2.empty())
{
    // …
}
```

If either mat is empty, the blend for that frame is skipped (leaving the original first frame intact). Any detailed error logging for empty loads (previously present as commented `cout` or `myofstream` statements) remains commented out in the x64 path .

### Alpha Computation

Before blending, the code converts percentage weights to normalized alpha values:

```cpp
float falpha_first  = fpercent_first  / 100.0f;
float falpha_second = fpercent_second / 100.0f;
```

These two alphas always sum to `1.0f`, ensuring a full dissolve between the two images.

### Blending with cv::addWeighted

OpenCV’s `addWeighted` performs the per‐pixel linear interpolation:

```cpp
cv::Mat dst;
cv::addWeighted(src1, falpha_first,
                src2, falpha_second,
                0.0f,
                dst);
```

Here, each output pixel is computed as:

```plaintext
dst(y,x) = src1(y,x) * falpha_first + src2(y,x) * falpha_second
```

The final `0.0f` offset preserves exact linear blending without additional scalars.

### Writing the Output

The blended result `dst` overwrites the first segment frame on disk:

```cpp
cv::imwrite(finalnewfilename_first.c_str(), dst);
cout << finalnewfilename_first << endl;
```

This both updates the on‐disk frame sequence and logs the written filename to the console.

---

By leveraging OpenCV’s fast in‐memory routines, the x64 implementation avoids external process calls, yielding lower overhead and tighter integration for real‐time crossfading of fractal‐trace animation frames.
