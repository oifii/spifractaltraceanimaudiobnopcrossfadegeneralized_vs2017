## 4. Crossfading Between Segments (Smoothing Visual Transitions)

### 4.3 32-bit / ImageMagick Notes (Historical/Reference Behavior)

#### Overview

In Win32 (x86) builds, crossfading between adjacent frame segments was originally implemented by invoking ImageMagick’s `convert` tool with a blend compose. This legacy code path is now commented out in favor of an OpenCV-based implementation on x64, but remains as a reference for historical behavior and for users maintaining 32-bit builds. The approach dissolves one clip into the next by looping through a fixed number of frames and varying blend percentages.

#### ImageMagick Blend Script Reference

A typical ImageMagick crossfade loop uses the `-compose blend` operator with `-define compose:args=P1,P2`, where `P1` is the percent opacity of the first frame and `P2` is the percent opacity of the second frame. For example, crossfading 5 frames between `frame_000117.jpg` and `frame_000270.jpg` can be scripted as follows :

```bash
cd "C:\Program Files (x86)\ImageMagick-7.0.8-Q8"
convert frame_000117.jpg frame_000270.jpg -compose blend -define compose:args=0,100   -composite frame_117-270_100-0pc.jpg
convert frame_000117.jpg frame_000270.jpg -compose blend -define compose:args=25,75  -composite frame_117-270_75-25pc.jpg
convert frame_000117.jpg frame_000270.jpg -compose blend -define compose:args=50,50  -composite frame_117-270_50-50pc.jpg
convert frame_000117.jpg frame_000270.jpg -compose blend -define compose:args=75,25  -composite frame_117-270_25-75pc.jpg
convert frame_000117.jpg frame_000270.jpg -compose blend -define compose:args=100,0  -composite frame_117-270_0-100pc.jpg
```

Conceptual mapping per crossfade step:

| Frame Index | Blend Args (First, Second) |
| --- | --- |
| 1 | 0%, 100% |
| 2 | 25%, 75% |
| 3 | 50%, 50% |
| 4 | 75%, 25% |
| 5 | 100%, 0% |

- `global_imagemagickconvertpath` was set to the ImageMagick `convert` executable path.

- The final command line echoes and logs the command before executing via `system()`.
