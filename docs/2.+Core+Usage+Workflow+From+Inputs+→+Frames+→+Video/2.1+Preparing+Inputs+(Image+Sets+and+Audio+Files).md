# Core Usage Workflow: From Inputs → Frames → Video

## 2.1 Preparing Inputs (Image Sets and Audio Files)

Before any frames are generated, the application must discover and register all source images and the one audio file to drive the animation. This involves:

### 2.1.1 Image Sets

- **Input expectations**

– A root folder (`global_imagefolder`) containing image files of a single extension (`global_imageextension`), e.g. `.jpg`, `.bmp`, `.png`.

– All images should share the same dimensions (later enforced during loading).

- **Discovery process**
- Construct a Windows DIR command that:
- Recurses subfolders (`/S`)
- Outputs only filenames (`/B`)
- Sorts by name (`/O:N`)
- Filters by the configured extension
- Redirects the list into `spifractaltrace_filenames.txt`
- Invoke the command via `system(...)`.
- Read each line of `spifractaltrace_filenames.txt` and append it to the `global_imagefilenames` vector.

- **Result**

`global_imagefilenames` is populated with full, alphabetically sorted paths to every input image.

```cpp
string quote = "\"";
string pathfilter = global_imagefolder + "\\*" + global_imageextension;
string systemcommand =
    "DIR " + quote + pathfilter + quote + "/B /S /O:N > spifractaltrace_filenames.txt";
system(systemcommand.c_str());

ifstream ifs("spifractaltrace_filenames.txt");
string temp;
while (getline(ifs, temp))
{
    global_imagefilenames.push_back(temp);
}
```

### 2.1.2 Audio File

- **Input expectation**

A single audio file path is provided (e.g. via command‐line argument) and stored in `global_audiofilename`.

- **Registration process**

The program simply pushes this path into the `global_audiofilenames` vector to unify handling (even though only one file is supported today).

```cpp
global_audiofilenames.push_back(global_audiofilename);
```

- **Result**

`global_audiofilenames` contains exactly one entry: the user-specified audio file. Downstream steps (beat, onset, pitch analysis) will reference this list to drive frame‐generation timing.

---

With these two vectors initialized, the pipeline is ready to load image data into memory and extract audio event timings in the next stages of the workflow.
